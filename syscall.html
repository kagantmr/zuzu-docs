Syscall descriptions

pquit

Terminate the calling process.

Arguments

RegisterNameDescription
r0statusExit status code stored in the PCB, retrievable by parent via wait

Returns

Does not return.

Errors

None. This syscall always succeeds.

Pitfalls

Do not use pquit to exit a thread unless you want the thread to terminate the entire process.

yield

Voluntarily reschedule the process away.

Returns

r0 = 0.

Errors

None. This syscall always succeeds.

pspawn

Create a frozen child process.

Arguments

RegisterNameDescription
r0namePointer to name string

Returns

0 on success.

Errors

CodeCondition
ERR_BADARGFailed in name copy
ERR_NOMEMFailed to create and fill PCB or handle vector full.
ERR_BADPTRString pointer is invalid.

Pitfalls

This syscall simply creates a FROZEN process that will not be scheduled. To actually fill the process address space, the init process must call asinject with parsed ELF data.

kickstart

Mark a FROZEN process as schedulable.

Arguments

RegisterNameDescription
r0argsPointer to argument struct

Returns

0 on success.

Errors

CodeCondition
ERR_BADARGFailed in struct copy or handle vector retrieval, or the task isn't frozen, or there's no such task.
ERR_BADPTRStruct pointer is invalid.

Pitfalls

Do not call kickstart on a process that does not have its address space filled with asinject, or you will get a prefetch abort and the process will be killed. Ordinary processes cannot call asinject, and if you're willing to spawn a child process, you must message the init process.

kill

Terminate a child process.

Arguments

RegisterNameDescription
r0handle_idxSlot index of task handle

Returns

0 on success.

Errors

CodeCondition
ERR_BADARGIf the handle doesn't exist, is not a process handle, or the process doesn't exist

Pitfalls

You cannot kill threads this way.
